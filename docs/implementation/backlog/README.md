# Qindu Backlog

This directory contains the canonical backlog and planning documents for the Qindu project.

## Files

| File | Description |
|------|-------------|
| `qindu-v1-backlog.yaml` | Canonical backlog (source of truth for all V1 work) |
| `qindu-v1-roadmap.md` | Bottom-up roadmap and macro dependency chain |
| `qindu-definition-of-ready.md` | Definition of Ready (DoR) - when an item can enter a sprint |
| `qindu-definition-of-done.md` | Definition of Done (DoD) - evidence required to close a sprint |
| `qindu-risk-register.md` | Cross-cutting risk register |
| `qindu-human-inputs.md` | Registry of human inputs required for backlog items |
| `qindu-sprint-template.md` | Template for sprint execution |

## Validation

- `qindu-v1-backlog.yaml` must be valid YAML
- All items must be 100% complete (gates, dependencies, acceptance evidence)
- ADR coverage must be verifiable
- Cross-references must be consistent
